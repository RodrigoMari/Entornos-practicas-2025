1a) HTML (HyperText Markup Language) es el lenguaje de marcado estándar que los navegadores web interpretan 
para estructurar documentos con texto, imágenes, hipervínculos, etc. 
Origen y creación: desarrollado por Tim Berners‑Lee en el CERN, entre 1989 y 1991, publicándose el primer 
borrador en 1993 con unas 18 etiquetas iniciales.

HTML 1.0 (década de 1991‑1993, ~18 etiquetas)
HTML 2.0 (1995, primer estándar del IETF)
HTML 3.2 (1997)
HTML 4.0

La ultima version es la HTML5 o la HTML5.2

1b) 
-Estructura semántica clara: uso apropiado de elementos
-Separación de presentación y contenido: evitar elementos de presentación
-Accesibilidad y compatibilidad internacional: incluir atributos alt, soporte para hojas de estilo, scripts, y 
utilizar codificación universal
-Validación y conformidad: seguir las reglas de conformidad, usar DTDs correctos, y hacer documentos validables

1c) Un elemento o atributo deprecated es aquel desalentado, no debe usarse en nuevos documentos, pero los navegadores 
deberían seguir soportándolo.

Un elemento obsolete se considera puramente histórico, no definido formalmente y no requiere soporte por 
parte del navegador (aunque en la práctica muchos aún funcionan).

1d) DTD (Document Type Definition) es una definición de la gramática SGML/HTML que formaliza la sintaxis 
permitida en un documento HTML. Se declara al inicio del archivo con <!DOCTYPE …>.


HTML 4.01 contempla tres variantes de DTD:
    Strict (no permite presentational markup),
    Transitional (permitiendo elementos y atributos deprecated para compatibilidad),
    Frameset (para documentos que usan <frameset>).

1e) Metadatos son datos sobre el documento web: autor, descripción, palabras clave, codificación, tipo de 
contenido, etc. Ayudan a motores de búsqueda, navegadores y herramientas a interpretar el contenido.
